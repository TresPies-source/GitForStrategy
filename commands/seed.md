# /seed

Access the seed pattern library. Seeds are reusable thinking patterns extracted from real experience.

## What This Does

1. Analyze the current context (what you're working on, what phase you're in)
2. Suggest relevant seed patches from the 13 available patterns
3. Apply the selected seed with a concrete checklist

## Available Seeds

**Core Seeds (10):**
- Three-Tiered Governance — structure that multiplies velocity
- Pace of Understanding — learn at depth, not speed
- Cost Consciousness — treat resources as constraints that sharpen thinking
- Voice Before Structure — read philosophy before writing artifacts
- Liberation Psychology — remove barriers to contribution
- Patient Learning — methodology prevents rabbit holes
- Era Architecture — plan across multiple releases
- Semantic Clustering — map what systems DO, not where files ARE
- Context Iceberg — 4-tier information architecture
- Spec-to-Prompt Pipeline — from specification to commission

**Field Seeds (3):**
- Portability Tiers — classify by dependency depth
- Agent Workspace — structured collaboration spaces
- Handoff Protocol — preserve context across agents

## How to Start

Ask the user: **"What are you working on right now?"**

Then use seed-library from `skills/patterns/seed-library/SKILL.md` to suggest and apply relevant seeds.

## Output

Applied seed guidance with a concrete checklist tailored to the current context.
