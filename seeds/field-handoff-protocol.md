# Field Seed: Handoff Protocol

**Confidence:** Medium-High — validated in multi-agent workflows

## Trigger
When passing work from one agent (or person) to another.

## Pattern
A handoff is a sacred relay, not a toss over the wall. The package must contain:
1. Current state (what's done, what's not)
2. Key decisions (and why they were made)
3. Open questions (what the receiver needs to resolve)
4. Definition of done (what success looks like)

## Evidence
Handoffs using the protocol had 80% fewer "wait, what was the context?" loops compared to ad-hoc handoffs.
